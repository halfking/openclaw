```bash

cd /Users/xutaohuang/workspace/ai/openclaw
node openclaw.mjs plugins install feishu

# 重启服务
./openclaw.mjs gateway stop
./start-with-private.sh gateway --port 18789
```

**结果：**

- ✅ 所有飞书工具已注册（5个核心工具）

- ✅ 事件调度器已就绪

## 🚨 关键问题与解决方案

### 问题1：配置兼容性

**问题：** 邮件配置不被openclaw支持

```json
"email": { ... }  // 导致启动失败
```

**解决方案：**

```bash
# 删除不支持的配置
node -e "
const fs = require('fs');
const config = JSON.parse(fs.readFileSync('~/.openclaw/openclaw.json'));
delete config.email;
fs.writeFileSync('~/.openclaw/openclaw.json', JSON.stringify(config, null, 2));
"
```

**预防措施：**

- 配置文件修改前进行版本兼容性检查
- 使用schema验证配置文件
- 在测试环境验证新配置

### 问题2：飞书插件缺失

**问题：** 插件目录为空，导致飞书功能不可用

**解决方案：**

```bash
# 重新安装插件
cd /Users/xutaohuang/workspace/ai/openclaw
node openclaw.mjs plugins install feishu

# 验证安装
ls -la extensions/feishu/
```

**预防措施：**

- 升级后验证插件安装状态
- 定期检查插件版本和功能
- 建立插件健康监控机制

### 问题3：服务启动冲突

**问题：** 多个进程监听同一端口

```
Port 18789 is already in use.
Multiple listeners detected.
```

**解决方案：**

```bash
# 正确停止服务
./openclaw.mjs gateway stop

# 检查端口占用
lsof -i :18789

# 强制清理
pkill -9 -f "openclaw.*gateway"
```

**预防措施：**

- 使用标准的停止命令而非强制终止
- 启动前检查端口状态
- 实现进程锁机制
